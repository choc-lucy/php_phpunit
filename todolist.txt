重要性とテスト容易性で判断

クラス設計
整数閉区間を示すクラス（あるいは構造体）をつくりたい。
整数閉区間オブジェクトは下端点と上端点を持つ

ロジック含まない → テスト容易性:高
- [x] 文字列表現を返す（例: 下端点 3, 上端点 8 の整数閉区間の文字列表記は "[3,8]"）。
  - [x] 2つの整数を引数に与えれば､決められたフォーマットの文字列を返す
  - [x] 整数以外の引数を与えたとき、エラーが返ってくる

重要性:低
ただし、
// 後で考える
上端点より下端点が大きい閉区間を作ることはできない。

テスト容易性:中
- [x] 整数の閉区間は引数で渡した整数を含むかどうかを判定できる。
  - [x] 引数で渡した整数が下端点よりも大きく､上端点よりも小さいことが判定できる

- [ ] Bの閉区間と等価かどうかを判定できる
  → Bの閉区間と下端点と上端点が等しいかどうかを判定できる
- [ ] Bの閉区間が完全に含まれるかどうかも判定できる
  → Bの閉区間の上端点がAの上端点よりも小さく､Bの閉区間の下端点がAの下端点よりも大きいかどうかも判定できる。
